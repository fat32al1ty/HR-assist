# Security Policy

## Supported versions

В активной разработке поддерживается только текущая `master` ветка.

## Reporting a vulnerability

Пожалуйста, не публикуйте уязвимость в публичных issues.

Отправьте приватное описание:

- шаги воспроизведения
- ожидаемое и фактическое поведение
- возможный импакт
- предложенный фикс (если есть)

До выхода патча просим соблюдать responsible disclosure.

## Минимальные правила безопасного деплоя

- Используйте сильный `JWT_SECRET_KEY`.
- Не храните секреты в Git.
- Используйте `AUTH_EMAIL_DELIVERY_MODE=smtp` в production.
- Регулярно ротируйте API-ключи.
- Ограничьте CORS только доверенными доменами.
