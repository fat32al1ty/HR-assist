# v0.12.0 — Phase 5.0 — Market-grounded resume audit + light Q&A onboarding

**Released:** 2026-04-25
**Phase plan:** [`.claude/skills/product-roadmap/phase-5.0-market-audit.md`](../.claude/skills/product-roadmap/phase-5.0-market-audit.md)

First step toward the AI-consultant thesis: replace the practice of a live career consultant with a market-grounded resume audit + lightweight onboarding clarification flow.

## What's new

### `/audit` page (frontend)

Four cards stacked on the new audit page, accessed automatically after resume upload (`/audit?resume_id={id}`):

- **Role Read** — how we parsed the resume: `role_family`, `seniority`, alt-roles with confidence scores. Surfaces "we read you as a Middle Backend Developer" so the user can correct it before everything downstream depends on it.
- **Market Salary** — predicted salary band for `role_family + seniority + location`, sourced from `salary_predictor` (LightGBM + median-by-role baseline fallback once `MIN_CORPUS=1000` is reached).
- **Skill Gaps** — top-5 skills that are required by ≥30% of vacancies for the user's role but absent from the resume. Each gap has a frequency bar showing how prevalent it is in matching market vacancies.
- **Resume Quality** — rule-based audit: missing English level when role demands it, empty experience highlights, missing GitHub for technical roles, etc. Severity ordering: `info < warn < error`.

### Light hybrid onboarding (Q&A clarification)

- 30 IT-specific question templates in `backend/app/data/onboarding_templates.yaml`, each with trigger conditions evaluated deterministically against the resume profile (e.g. `seniority_is_middle AND years_in_role_between_4_and_6` → "are you Middle or pre-Senior?").
- Trigger DSL supports `AND` / `OR` / `NOT` with standard precedence (AND tighter than OR).
- `feature_onboarding_llm_classifier_enabled = False` by default — pure rules-only path. LLM-based question selection is behind a feature flag and **applies `pii_scrubber.scrub_pii()` to canonical_text before sending to OpenAI** (Privacy Level A compliance).
- Up to 5 questions surfaced per resume; answers persist in `resume_clarifications` and are folded into future audits.

### Backend: audit engine

- New `resume_audits(id, resume_id, audit_json, prompt_version, computed_at, cost_usd)` table with `prompt_version="audit-v1"` and 7-day cache TTL (`resume_audit_cache_ttl_days=7`).
- Cost cap **$0.05/DAU/day** (`resume_audit_cost_cap_usd_per_day`) — when exceeded, audit falls back to template-only mode (no LLM normalization of skill names).
- Cost flows end-to-end: `_normalize_skills` → `_build_skill_gaps` → `compute_audit` → `cost_usd` column. Replaced `lru_cache` with manual `dict[str, tuple[str, float]]` so cost from cache hits is correctly attributed (0).

### Admin observability

- `GET /api/admin/overview` now reports `cost_p95_per_dau_usd` — 95th-percentile per-DAU LLM cost for resume audits over the last day. Surfaces cost-cap pressure before users hit the fallback.
- `GET /api/admin/audits/sample?limit=N` — sample of recent audits (id, resume_id, prompt_version, computed_at, cost_usd) for manual quality spot-checks.

### Eval bootstrap

- 20 self-labeled fixtures in `backend/tests/fixtures/audit_bootstrap/case_*.json` covering: clear seniority/role classifications, ambiguous middle-vs-senior, cross-domain noise, missing market signals, low-confidence cases.
- `tests/eval/test_audit_regression.py` — two parametrized suites: hard deterministic assertions (always run, no API key needed) + LLM-as-judge for soft signals (skipped if no OpenAI key, xfailed past $0.50 session cost cap).
- Cross-domain noise edge cases (`case_17`, `case_20`) where the LLM judge is empirically unstable are listed in `LLM_JUDGE_FLAKY_CASES` and xfail-with-strict=False if the judge happens to disagree — passes count, flakes don't break CI.
- `tests/eval/test_audit_cost_cap.py` and `tests/eval/test_onboarding_classifier_consistency.py` cover the cost-cap fallback and onboarding rule consistency.

## DB migrations

- `0028_resume_audit_cache.py` — `resume_audits` (`resume_id` UNIQUE, FK CASCADE, `audit_json JSON`, `prompt_version VARCHAR(32)`, `computed_at TIMESTAMP`, `cost_usd NUMERIC`).
- `0029_resume_clarifications.py` — `resume_clarifications` (`resume_id` + `question_id` UNIQUE, `answer_value`, `answered_at`).

## Feature flags introduced

| Flag | Default | Purpose |
|------|---------|---------|
| `feature_resume_audit_enabled` | `True` | Master switch for `/audit`. |
| `feature_resume_audit_template_mode_enabled` | `False` | Force template-only mode (skip LLM normalization). |
| `feature_onboarding_llm_classifier_enabled` | `False` | LLM-based question selection vs deterministic rules. |
| `resume_audit_cost_cap_usd_per_day` | `0.05` | Per-DAU daily cost cap. |
| `resume_audit_cache_ttl_days` | `7` | Audit cache age before recompute. |

## Decision-gate (next 14 days)

Per the eval design, this release is gated for 14 days under flag with three metric classes:

1. **Engagement:** `audit_view_rate` (% of resume-upload sessions that open `/audit`) ≥ 60%.
2. **Quality:** `audit_role_corrected_rate` (% of audits where the user accepts the role read without correction) ≥ 70%.
3. **Cost:** `cost_p95_per_dau_usd` ≤ $0.05 over a rolling 7-day window.

If 2 of 3 don't confirm, Phase 5.1 scope is reopened.
