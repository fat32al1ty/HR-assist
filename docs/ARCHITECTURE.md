# Architecture Overview

## Цель

Сервис помогает пользователю:

1. Загрузить резюме.
2. Получить структурный профиль кандидата.
3. Найти релевантные вакансии по смыслу, а не только по ключевым словам.

## Компоненты

- `frontend/` - Next.js UI (личный кабинет, загрузка резюме, подбор вакансий).
- `backend/` - FastAPI API и бизнес-логика.
- `postgres` - источник истины по пользователям, резюме, вакансиям, feedback.
- `qdrant` - эмбеддинги резюме/вакансий и семантический поиск.
- `openai` - анализ текстов и LLM/embedding слой.

## Поток данных

1. Пользователь регистрируется и подтверждает email.
2. Загружает PDF/DOCX.
3. Backend извлекает текст и запускает LLM-анализ.
4. Профиль резюме сохраняется в PostgreSQL и эмбеддится в Qdrant.
5. Источники вакансий пополняют внутреннюю базу.
6. Matching-сервис ранжирует вакансии и возвращает подборку в UI.
7. Лайки/дизлайки записываются в PostgreSQL и используются для улучшения поиска.

## Security baseline (текущее состояние)

- JWT авторизация.
- Обязательная верификация email.
- Обязательный beta-key при регистрации.
- Второй код при входе (email OTP).
- Guard для prompt-injection на untrusted-тексте резюме и вакансий.

## Следующий уровень для production

- Rate limiting и anti-bruteforce.
- Централизованные audit logs.
- Secret manager вместо env на боевом контуре.
- Регулярные security-тесты и SAST/DAST в CI.
